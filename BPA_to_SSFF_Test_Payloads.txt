TEST PAYLOAD 4: INVALID FIELD FORMATS
================================================================================

Scenario:
  Payload with fields in incorrect data types or invalid formats.
  Tests data type validation and format conversion robustness.

Request Payload:
{
  "code": "POS-2025-00145",
  "effectiveStartDate": "06-15-2025",
  "effectiveStatus": "ACTIVE",
  "company": "COMP_001",
  "department": "DEPT_SALES_001",
  "changeReason": "INVALID_REASON",
  "targetFTE": "1.5X",
  "regularTemporary": "UNKNOWN",
  "vacant": "yes",
  "cust_standardHours": "forty",
  "parentPosition": {
    "code": 12345
  }
}

Issues in payload:
  - effectiveStartDate: Invalid format "06-15-2025" (expected YYYY-MM-DD)
  - changeReason: Invalid enum value "INVALID_REASON"
  - targetFTE: String "1.5X" instead of numeric 1.5
  - regularTemporary: Invalid enum "UNKNOWN" (expected R/T)
  - vacant: String "yes" instead of boolean false
  - cust_standardHours: String "forty" instead of numeric 40
  - parentPosition/code: Numeric 12345 instead of string "POS-XXXX-XXXXX"

Expected CPI Behavior:

  1. Enrichment wrapping succeeds (no type validation at this stage)
  2. JSON to XML conversion succeeds (XML allows mixed types)
  3. BPA to SSFF Mapping applies data type conversions:
     - Date format conversion fails: "06-15-2025" → DateTime parse exception
     - Mapping detects invalid date format
     - Exception: "Invalid date format for effectiveStartDate"
  4. Mapping error caught → Alternatively:
     - If conversion is attempted on numeric fields (targetFTE, hours):
       - String "1.5X" and "forty" cannot convert to number
       - NumberFormatException logged
  5. Mapping validation fails
  6. Error message returned to BPA
  7. Error End Event triggered

Expected HTTP Response:
  Status Code: 400 Bad Request
  
  {
    "error": {
      "message": "Data type conversion error during mapping",
      "details": "Invalid date format for effectiveStartDate: Expected YYYY-MM-DD, got 06-15-2025",
      "field": "effectiveStartDate",
      "integrationFlowName": "Create Position from BPA to SSFF"
    }
  }

Alternative if OData receives request:
  Status Code: 200 OK
  
  {
    "UpsertResponses": {
      "PositionUpsertResponse": {
        "status": "ERROR",
        "code": "POS-2025-00145",
        "errorMessage": "Invalid field format: targetFTE must be numeric, received 1.5X"
      }
    }
  }

Monitoring Observations:
  • Message processing time: <2 seconds (fails during mapping)
  • Mapping logs: "Type conversion failed for field targetFTE"
  • Error End Event triggered
